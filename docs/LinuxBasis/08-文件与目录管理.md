## 常见文件操作
市面上的教程已经介绍得很完备了，我只进行补充，或者讲一些容易弄错的地方。
### 复制目录
```bash
cp -r <source_dir> <dest_dir>
```
假设：`<source_dir>`可以表示为`/dirname1/basename1`。那么：
- 当`<dest_dir>`不存在时，复制后得到的新目录到底是`<dest_dir>`本身（在`<dest_dir>`看到的是`<source_dir>`目录下的内容）。
- 当`<dest_dir>`存在时，复制后得到的新目录是`<dest_dir>/basename1`（在`<dest_dir>`看到的是名为`basename1`的目录）。


### 文件传输`scp`
`scp`命令基于SSH协议，用于在本地和远程主机之间传输文件。

`username@remote`中，`username`是远程主机的用户名，`remote`是远程主机的IP地址。执行`scp`命令时，需要输入远程主机的密码。
#### 上传
```bash
scp /local/path/to/file username@remote:/path/to/file
```
#### 下载
```bash
scp username@remote:/path/to/file /local/path/to/file
```

### 压缩与解压 tar
归档实用程序。  通常与压缩方法结合使用，例如 `gzip` 或 `bzip2`。  更多信息：https://www.gnu.org/software/tar/manual/tar.html。

#### 压缩
- 创建存档并将其写入文件：    
    ```bash
    tar - cf 目标文件.tar 路径/到/文件1 路径/到/文件2 ...
    ```
- 创建一个 gzip 压缩文件并将其写入文件：    
    ```bash
    tar - czf 目标文件.tar.gz 路径/到/文件1 路径/到/文件2 ...
    ```
- 使用相对路径从目录创建一个 gzip 压缩文件：    
    ```bash
    tar - czf 目标文件.tar.gz --directory 路径/到/目录 .
    ```
- 创建压缩存档并将其写入文件，使用文件扩展名自动确定压缩程序：    
    ```bash
    tar - caf 目标文件.tar.xz 路径/到/文件1 路径/到/文件2 ...
    ```
#### 解压
以下操作，解压后压缩包依然存在。
- 解压到 `当前目录/文件名`目录中：    
    ```bash
    tar -xvf 文件名.tar[.gz|.bz2|.xz]
    ```
- 解压到目标目录中：    
    ```bash
    tar -xf 文件名.tar[.gz|.bz2|.xz] --directory 目标目录
    ```
#### 非解压式读取
- 详细列出 tar 文件的内容：    
    ```bash
    tar - tvf 源文件.tar
    ```
- 从存档文件中提取与模式匹配的文件：    
    ```bash
    tar - xf 源文件.tar --wildcards "*.html"
    ```







## 回收站
Linux本身没有直接支持回收站的功能，可以通过`trash-cli`工具来实现。当然可以通过改变`rm`命令的行为来实现，但个人认为，最好不要改变`rm`这样系统的内置命令。
```bash
sudo apt install trash-cli
```
### 移动到回收站
```bash
trash path/to/file
```
### 列出回收站中所有的文件
```bash
trash-list
```
会显示文件原路径，示例：
```bash
username@DeviceName:~$ trash-list
2025-04-03 21:31:55 /home/username/sources.list
2025-04-03 21:31:54 /home/username/source.tar.gz
```
### 还原
1. 还原回收站中的某个文件，或者原目录是所给目录的文件
    ```bash
    trash-restore original/path/to/file/or/directory
    ```
2. 还原原目录在当前目录下的文件
    ```bash
    trash-restore
    ```
    示例：
    ```bash
    username@DeviceName:~$ trash-restore
    No files trashed from current dir ('/home/username')
    ```
### 永久删除
1. 永久删除回收站中的某个文件，或者目录是所给目录的文件 
    ```bash
    trash-rm path/to/file/or/directory
    ```
2. 永久删除匹配所给正则表达式（Regex）的文件
    ```bash
    trash-rm <Regex>
    ```
3. 永久删除回收站中的所有文件
    ```bash
    trash-empty 
    ```
4. 永久删除回收站中早于10天前的文件
    ```bash
    trash-empty 10
    ```
